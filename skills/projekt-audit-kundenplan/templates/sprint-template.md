# Sprint-Template

Verwende dieses Template für jeden neu generierten Sprint-Plan in `sessions/sprints/SPRINT_<THEMA>.md`. Sektionen mit `<…>` sind Platzhalter, die durch konkrete Inhalte ersetzt werden müssen. Nicht zutreffende Sektionen können entfallen.

---

```markdown
# Sprint: <Thema>

**Erstellt:** YYYY-MM-DD
**Status:** OFFEN
**Voraussetzung:** <vorheriger Sprint oder konkrete Vorbedingung>
**Referenz:** <Pfad zum auslösenden Audit-Dokument>

---

## Ziel

<1–3 Sätze: was am Ende anders ist als vorher>

---

## Phase 1 — <Phasen-Titel>

### <ID>-1: <Task-Titel>

**Ziel:** <1 Satz>
**Datei(en):** <konkrete Pfade>
**Aufwand:** <Spanne, z. B. 2–4 Stunden>

- [ ] <konkreter Schritt>
- [ ] <konkreter Schritt mit Beispielcode bei Bedarf:>
  ```bash
  <befehl>
  ```

### <ID>-2: <Task-Titel>

…

---

## Phase 2 — <Phasen-Titel>

…

---

## Definition of Done

- [ ] <messbares Kriterium 1>
- [ ] <messbares Kriterium 2>
- [ ] <messbares Kriterium 3>
```

## Pflicht-Eigenschaften

- Jede Task hat eine **konkrete Datei** oder einen **konkreten Befehl**
- Jede Task hat eine **Aufwand-Spanne** (nicht-Punktschätzung)
- Jede Phase endet mit einem **messbaren Ergebnis**
- Definition of Done ist **prüfbar** (kein „funktioniert", sondern „Test X grün")
- Keine Marketing-Sprache, keine Emojis
- Code-Beispiele in fenced blocks mit Sprache
